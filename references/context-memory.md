# 上下文压缩与记忆强化协议

> 目标：在长对话、上下文被压缩后，不丢失关键信息、不跑偏任务。
> 适用：DeepSeek-V4-Pro / Xiaomi-MiMo-V2.5-Pro。

---

## 1. 核心循环：压缩 → 重读 → 确认

```
长对话进行中
   ↓
上下文接近上限 → 压缩前提炼关键要点（State Snapshot）
   ↓
上下文被压缩
   ↓
新一轮开始 → 重新读取要点（Re-anchor）
   ↓
对照最近文件/输出复核 → 确认当前所处步骤
   ↓
继续工作（记忆已加深）
```

---

## 2. 压缩前：提炼状态快照（State Snapshot）

在上下文将被压缩前，把以下信息提炼成结构化要点：

- **目标（Goal）**：用户真正想达成什么。
- **已决策（Decisions）**：已经定下来的方案、选型、约定。
- **进度（Progress）**：多步任务中，已完成哪些、当前在第几步。
- **未决问题（Open Questions）**：还没解决/还没确认的点。
- **关键路径（Key Paths）**：涉及的关键文件、函数、命令。
- **约束（Constraints）**：用户提的限制、偏好、禁止项。

快照要短、结构化、可快速重读。

---

## 3. 压缩后 / 新轮开始：重新锚定（Re-anchor）

1. **重读**上一步的状态快照。
2. **核对**：对照最近的文件状态、命令输出，而非仅凭记忆，确认快照仍准确。
3. **定位**：明确"我现在在多步任务的哪一步"。
4. **验证引用**：快照里提到的文件/函数/参数，使用前确认它当前还存在。
5. 锚定完成后再继续推理与执行。

---

## 4. 加深记忆的技巧

- **复述关键信息**：新轮开始时，用一两句话复述目标与进度，形成记忆强化。
- **以文件为真相源**：代码、配置、输出是事实来源；记忆只是索引，冲突时以实际文件为准。
- **增量更新快照**：每完成一个里程碑，更新快照，而不是等到压缩才整理。
- **链接关联信息**：把相关的决策、文件、问题关联起来，便于整体重建上下文。

---

## 5. 防跑偏检查

每次重新锚定后问自己：

- [ ] 我还记得用户真正的目标吗？
- [ ] 我现在做的事，和目标一致吗？
- [ ] 有没有因为上下文丢失而重复或遗漏的步骤？
- [ ] 快照里的关键文件/函数，现在还存在、还是这个状态吗？

---

## 6. 与宿主机制配合

- 在 Claude Code / Codex 中，上下文会被宿主自动压缩并在下一窗口提供摘要——
  本协议是在此之上额外做"主动提炼 + 主动重读 + 对照复核"，进一步降低信息丢失。
- 不依赖单一记忆来源：宿主摘要 + 本快照 + 实际文件，三者交叉确认。
