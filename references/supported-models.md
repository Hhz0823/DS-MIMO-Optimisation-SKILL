## 4. 如何为某个模型启用

1. 选择对应宿主的安装方式（见 `INSTALL.md` 或一键脚本 `install.sh` / `install.ps1`）。
2. 确保该模型在宿主中被设为当前模型。
3. 确保宿主把 `SKILL.md`（及按需的 `references/`）作为系统/项目指令加载。
4. 用"写得乱 + 需推理/写代码"的消息验证行为。

---

## 5. 模型差异提示

- **推理型模型**（DeepSeek-R1、Kimi 思考版等）：本 SKILL 的"多元化/逻辑/深度推理"
  与其原生思考链叠加，注意避免重复冗长——可在外显推理时让协议只做"结构约束"。
- **长上下文模型**（Kimi、Qwen 长文版）：优先用"引用代替粘贴 + 分层加载"，
  把长上下文用在真正需要的内容上。
- **小参数/快速模型**：可只加载 `SKILL.md` 摘要，不强制读全部 references，降低开销。
